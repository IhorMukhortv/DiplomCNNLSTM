# План впровадження: Генерація дипломної роботи у формат Word (DOCX) через Pandoc

Цей план передбачає перехід від кастомного парсера Markdown на основі `python-docx` до використання утиліти **Pandoc** для конвертації розділів дипломної роботи у формат `.docx`. Це дозволить отримати нативні формули MS Word (у форматі OMML/Equation), які виглядають професійно та підтримують редагування, а також спростить парсинг складних елементів (таблиць, списків, посилань). 

Для дотримання вимог ДСТУ (шрифти, поля, абзаци, центрування формул із номерами праворуч) буде використано гібридний підхід: генерація файлу через Pandoc з подальшою автоматичною пост-обробкою стилів за допомогою `python-docx`.

> [!IMPORTANT]
> **Необхідність встановлення Pandoc:**
> Для роботи нового скрипта компіляції на локальній машині користувача має бути встановлено утиліту Pandoc. На ОС Windows її можна встановити однією командою:
> ```powershell
> winget install JohnMacFarlane.Pandoc
> ```
> Скрипт автоматично перевірятиме наявність Pandoc у системі та виводитиме зрозумілу помилку з інструкцією, якщо утиліту не знайдено.

> [!NOTE]
> **Гібридний підхід (Pandoc + python-docx):**
> 1. Об'єднання всіх 11 розділів Markdown в один тимчасовий файл `temp_thesis.md`.
> 2. Запуск Pandoc для конвертації `temp_thesis.md` у вихідний `diploma_work.docx`.
> 3. Запуск пост-процесора на основі `python-docx`, який відкриває згенерований документ і:
>    * Налаштовує поля сторінки (ліве 25 мм, верхнє/нижнє 20 мм, праве 15 мм).
>    * Застосовує до стилю `Normal` шрифт Times New Roman 14 пт, інтервал 1.5 та абзацний відступ 1.25 см.
>    * Знаходить формули (блоки омакетів `m:oMath` / `m:oMathPara`) та центрує їх, а номери формул (наприклад, `(8.1)`) вирівнює по правому краю за допомогою нативних таб-стопів Word.
>    * Центрує рисунки та підписи до них.
>    * Додає рамки та відступи до таблиць.
> 4. Видалення тимчасового файлу `temp_thesis.md`.
