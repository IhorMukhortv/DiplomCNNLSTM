# Написання та структурування дипломної роботи

Ця навичка допомагає структурувати розділи дипломної роботи, формулювати наукову новизну, практичне значення, розписувати математичні моделі та структурувати результати експериментів.

## Коли використовувати навичку

- Розробка плану та структури дипломної роботи.
- Написання вступу (мета, завдання, об'єкт, предмет, наукова новизна).
- Написання теоретичного розділу (огляд існуючих методів прогнозування генерації ФЕС).
- Опис математичного апарату (шари згортки CNN, механізм LSTM, функції втрат).
- Структурування експериментальної частини (результати навчання, порівняння з іншими моделями, графіки).
- Написання висновків та підготовка списку літератури.

## Рекомендована структура дипломної роботи

1. **Вступ:**
   - **Актуальність теми:** Зростання частки ВДЕ, нестабільність сонячної генерації, необхідність балансування енергомережі.
   - **Мета роботи:** Розробка комп'ютерно-інтегрованої системи короткострокового прогнозування генерації ФЕС з підвищеною точністю.
   - **Об'єкт дослідження:** Процес короткострокового прогнозування генерації електроенергії на фотоелектричних станціях.
   - **Предмет дослідження:** Гібридні нейромережеві моделі архітектури CNN-LSTM та алгоритми обробки даних часових рядів.
   - **Наукова новизна:** Модифікація архітектури або комбінація вхідних факторів для підвищення точності прогнозу за несприятливих погодних умов.
   - **Практичне значення:** Можливість інтеграції розробленого програмного забезпечення з системами диспетчеризації ФЕС.

2. **Розділ 1. Аналіз сучасного стану систем прогнозування генерації ФЕС:**
   - Огляд фізичних, статистичних та машино-навчальних методів (ARIMA, SVM, Random Forest).
   - Обґрунтування переходу до глибокого навчання (Deep Learning).
   - Аналіз переваг та обмежень гібридної архітектури CNN-LSTM.

3. **Розділ 2. Математичне та програмне забезпечення гібридної моделі CNN-LSTM:**
   - Формальний опис шарів одновимірної згортки (Conv1D) для аналізу погодних ознак.
   - Опис роботи осередку LSTM (Forget gate, Input gate, Output gate) для збереження часової залежності.
   - Схема конвеєра даних (попередня обробка, нормалізація, формування ковзного вікна).

4. **Розділ 3. Архітектура та програмна реалізація комп'ютерно-інтегрованої системи:**
   - Проектування бази даних часових рядів (PostgreSQL + TimescaleDB).
   - Розробка асинхронного REST API на FastAPI для надання прогнозів.
   - Опис взаємодії модулів системи (збір даних, оновлення бази даних, інференс нейромережі).

5. **Розділ 4. Експериментальні дослідження та аналіз результатів:**
   - Опис набору даних (dataset) та умов експерименту.
   - Аналіз результатів прогнозування за метриками MAE, RMSE, MAPE.
   - Порівняльний аналіз (CNN-LSTM порівняно з базовими моделями LSTM та CNN окремо).

6. **Висновки:**
   - Стислий звіт за кожним завданням роботи.
   - Оцінка досягнення мети дослідження.

## Академічний стиль та стандарти

- **Точність формулювань:** Уникайте розмовного стилю та надмірної метафоричності. Використовуйте безособові або пасивні конструкції («було розроблено», «досліджено», «запропонована модель»).
- **Оформлення формул:** Кожна математична формула повинна мати пояснення всіх її змінних безпосередньо під нею.
- **Джерела:** Кожне твердження про властивості чи результати інших авторів повинно підкріплюватися посиланням на список літератури.
